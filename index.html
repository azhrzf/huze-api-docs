Huze Api Documentation

The Huze API is a web service that provides access to various functionalities of the Huze project. It utilizes bearer token authentication for secure access to the API endpoints. To access the Huze API, you need to obtain a bearer token by authenticating with valid credentials. The token should be included in the Authorization header of each API request.

The Huze API supports the following functionalities:

  1. User Management: Allows users to register, log in, and manage their profile information.
  2. Post Management: Enables users to create, retrieve, update, and delete posts.

To interact with the Huze API, send HTTP requests to the appropriate endpoints, providing the required parameters and including the bearer token in the request headers. The API responds with JSON data containing the results or error messages.

Ensure that you include the bearer token in each request's header to authenticate your requests properly and gain access to the authorized functionalities. Refer to the API documentation for detailed information on the available endpoints, request parameters, and response formats. Please note that unauthorized access or misuse of the Huze API may result in restrictions or account suspension.

Base URL

  1. The base URL for all API endpoints is: https://huze-management.et.r.appspot.com/v1
  2. The base url for processing the model and using machine learning is: https://backend-flaskh5-nyighyk53q-et.a.run.app
